PIPS (Planetary Image Publication Server)

Code »

Summary

The Planetary Image Publication Server is a publication tool for researcher that reads complex data out of cube files and allows users faster access to the data than ever before. It also allows the data to be used in automated photo editing actions. Researchers can add north arrows and scale bars to an image at the click of a button instead of learning complex function calls for it. This server takes the power of ISIS3 and simplifies it so that researchers of any experience level can use it.


Challenges

The largest Challenges that I faced were in learning new technology. This server is written in Node.js which is a language a framework that I had very little experience in. I learned how to make basic Node.js servers is Advanced Web Programming but I had never tried working synchronously in Javascript. I had to teach myself how to make Promises and use async functions which I had never done before. Since the server needs to run ISIS commands on a command line, I had to create async functions for each ISIS command and make sure that the functions where finished before responding to the user.

Design

The design of this project was flipped on its head when our team decided to switch from Flask to Node.js. We made the switch of frameworks based on the image processing ability that Javascript has to offer as well as overall performance and portability boosts. The hardest part was synchronizing user instances but with the help of user cookies the server can handle many requests at once while never mixing data instances.
After perfecting the server instances and all the photo editing capabilities using Scalable Vector Graphics the product will be available on a server space at the USGS office and it will also be downloadable as a Docker Image for any researchers or companies that would like a server of their own.
