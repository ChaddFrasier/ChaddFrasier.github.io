Summary

This is a project that was assigned to us in my Operating Systems class where we wrote a simulation operating system in C using hand written utility functions and a hand written memory block system. This project took 16 weeks in class to complete and I plan on rewriting some of the components for efficiency. I really enjoyed solving the problems that were presented in this project. The challenges ranged from something as simple as learning how to correctly write text to files in C, while other challenges were much more difficult like learning multi-threading.

Challenges

The challenges in this project proved to be very useful in other scopes. I learned how to program simple yet useful utility style functions and learning them in C helps me to translate the knowledge into other languages with ease. In this project I learned how to use the gcc make functionality to help compile and link all the files. Learning C also helped me to teach myself how to program in C++, I've now written a Doubly Linked List class in C++ as well as a few sorting algorithms.

Design

The system starts with a configuration file that tells the system what CPU scheduling algorithm to use and it also passes the name of the metadata file and the for the batch run of the system. The metadata contains lines that correspond to CPU actions that are being performed. After the system is configured the system opens the metadata file and creates a new PCB for each action in the file. Each PCB gets placed into a modified linked list and then is sorted by the CPU scheduler before being executed. When the system begins to run the process blocks are ticked down by a pre-configured amount until there is no more time left on the process. It is then removed from the list and the system keeps going. The system will halt when there is no more process control blocks in the list. The CPU scheduler needs to run whenever the system loops to make sure the block are still in the correct order. When a memory action is found the memory management unit takes control of the allocating and accessing of memory. The MMU is in charge of allocating memory blocks and throwing errors when an access is attempted in a forbidden block. All outputs get logged by the logging system that either outputs to console, or outputs to file. The logging system tells every action that is executed for the system instance.
